ANDROID KEYSTORE INFORMATION
============================

For Google Play Store deployment, you'll need to create a signing keystore.

KEYSTORE GENERATION COMMAND:
keytool -genkey -v -keystore lemur-express-release.keystore -alias lemur-express -keyalg RSA -keysize 2048 -validity 10000

SUGGESTED KEYSTORE DETAILS:
- Keystore Name: lemur-express-release.keystore
- Alias: lemur-express
- Password: [Create a strong password and keep it secure]
- Validity: 10000 days (about 27 years)

CERTIFICATE INFORMATION TO PROVIDE:
- First and Last Name: Lemur Express 11 Development Team
- Organizational Unit: Mobile Development
- Organization: Lemur Express 11
- City: Palacios
- State: Texas
- Country Code: US

IMPORTANT SECURITY NOTES:
1. Keep the keystore file and passwords extremely secure
2. Store backups in multiple secure locations
3. Never commit keystore files to version control
4. The same keystore must be used for all app updates
5. Losing the keystore means you cannot update the app

GRADLE CONFIGURATION:
Add to android/app/build.gradle:

android {
    signingConfigs {
        release {
            storeFile file('path/to/lemur-express-release.keystore')
            storePassword 'your-keystore-password'
            keyAlias 'lemur-express'
            keyPassword 'your-key-password'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'
        }
    }
}
